   - In simple Data Binding is Simply and Communication between TypeScript file to html.

Types of DataBinding
 - OutputData.
	1. String Interpolation ( {{ data }} )

 - React to Events

Combination of Above two bindings.

String Interpolation


Syntax

Used

Property Binding


Property Binding
	- Its the Same concept of binding data from typescript file to html but here we are binding the Actual Value such as boolean or other data types.
	- In Property binding we are passing Actual value Such as True or false.

Example
	-	< input type="text" value="Hari" >        $0.getAttribute('value')      $0.value

Attribute vs Property
	- As you seen after overriding the value the $0.value changes and $0.getAttribute('value') remains constant.
	- This is Because Attribute Initializes Dom property and then they are Done.we can't Change Attribute After Initializing.
	- Attributes are defines by the HTML where as property is defines by the DOM (Document Object Model)
	- So we Can Change DOM that means Property value also we can change
	- With Property Binding in Angular we are Binding property to Dom Model

Example
	- < input id="{{nameID}}" type="text" value="Hari" >			nameID = 'ID'
	- < input [id]="nameID" type="text" value="Hari" >
	

	- < input disabled = {{'false'}} id="{{myID}}" type="text" value="Hari" >
	- < input [disabled] = "false" id="{{myID}}" type="text" value="Hari" >

Example 
	- < button class="btn btn-primary" > < /button >
	- < button class="btn btn-primary" [disabled]="!allowNewServer" >Add Server </button >
	- < p [innertext]="allowNewServer" > < /p >

TypeScript
	- constructor(){
		setTimeout(() =>{
		   this.allowNewServer = true;
		},2000)
	  }
	- ngOnInit(){
	  }

Use
	- When we are going to Display Something in UI or Print Something Use String Interpolation
	- When we want to change some property on behalf of and html file use Property Binding

Component Interaction


Component Interaction
    - All this while we are working on our Dev component which is nested inside our root component
    - App component is called our parent component and our test component is called child component
    - Now as you seen in previous videos out component is working independently to each other.
    - But while working with in project we come up with a scenario where our component need to communicate with each other.
    - Then how they communicate with each other.
    - they communicate with each other with the help of @input()(parent---->child) and @output()(child---->parent) decorator
    - With Input decorator the child can accept the input from parent
    - With output decorator the child will send out data or output to parent
    - So here we are Simply going to pass name from appcomponent(parent) to devcomponent(child)
    - And we are also passing Hi Codeme! from devcomponent(child)  to appcomponent(parent)


 Html (Parent)
	< app-dev [parentData]="name" > < /app-dev >

 TypeScript (Parent)
	name = 'Hari'

 TypeScript (child)
	@Input() parentData;  			or   			@Input('parentData') name;

 Html(child)
	< h4 > {{parentData}} < /h4 >			or			< h4 > {{name}} < h4 >


	-Here we have successfully completed to sending data from parent component to child component
	- So let start with send data from child component to parent component
	- this is Slightly different because in parent component html we have a child component selector so we can easily bind data in this way.
	- But in child component we don't have parent component selector so we can't send the data in same way.
	- The Way child component send data to parent component is using Events
	
	
 TypeScript (child)
	- We are going to create object for eventemitter class and also importing same.

	- @Output() childEvent = new EventEmitter()
	- After that Simply we are going to fire this event on Button Click

	- fireEvent(){
		this.childEvent.emit('Hey Codeme');
	  }

 Html (child)
	< button (click)="fireEvent()" > Send Event < /button >			
    After event is Emitted on button Click so we need to capture this on button Click

 Html (Parent)
	- Just like how we bind to click event we can bind to the custom event as well
	< app-text (childEvent)="" [parentData]="name" > < /app-dev >		
    
    Once the Event is capture now assign the to the property called message to $event variable
